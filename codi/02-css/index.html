Jerarquia de Css - Restricciones

Los navegadores aplican estilos a los documentos html por jerarquia basada en:

  1. La Cascada
  2. La especificidad
  3. La Herencia

La cascada es el procedimiento con el que un navegador aplica los estilos y el orden de importancia o restriccion es el siguiente:

  1. Origen del codigo
    1. Reglas del User Agent
    2. Reglas del Usuario
    3. Reglas del desarrollador
  2. La especificidad del selector
  3. Orden de Aparicion

La especificidad es la fuerza restrictiva o peso de un selector.

  • los estilos en linea 1
  • Los iden tificadores "id" 0,1
  • clases, atributos, pseudoclases 0,0,1
  • Las etiquetas 0,0,0,1
  • !important - Salta toda la Jerarquia

La Herencia es la capacidad que tienen los elementos padre de hacer heredar a sus hijos sus propiedades. Valores iherit o heredados

Resetear navegadores

Ante la diferente interpretación del aspecto de los elementos de html que utiliza cada navegador, nace la necesidad de borrar o resetear todas estas propiedades nativas de los navegadores para dejar el control a los desarrolladores.
Existen en el mercado diversos archivos css que funcionan como reseteadores estandard, cómo normalize.css o reset.css.
